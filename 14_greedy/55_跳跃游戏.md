# 55. 跳跃游戏

## 题目
给你一个非负整数数组 nums ，你最初位于数组的 第一个下标 。数组中的每个元素代表你在该位置可以跳跃的最大长度。

判断你是否能够到达最后一个下标。如果可以，返回 true ；否则，返回 false 。

示例:
- 输入: nums = [2,3,1,1,4]
  输出: true
  解释: 可以先到达下标 1（跳 1 步），再从下标 1 跳 3 步到达最后的下标 4
  
- 输入: nums = [3,2,1,0,4]
  输出: false
  解释: 无论如何，你总会到达下标 3，其最大跳跃长度是 0，因此无法到达最后的下标 4
  
- 输入: nums = [0]
  输出: true

## 思路
使用**贪心算法**，维护能到达的最远位置：
1. 维护一个变量 max_reach，表示当前能到达的最远位置
2. 遍历数组，对于每个位置 i：
   - 如果 i > max_reach，说明无法到达此位置，返回 false
   - 否则，更新 max_reach = max(max_reach, i + nums[i])
   - 如果 max_reach >= n-1，说明已能到达最后位置，返回 true
3. 贪心策略：每个位置选择能跳最远的方式

关键：不需要记录具体的跳跃路径，只需维护能到达的最远距离

## Python 代码

```python
def canJump(nums):
    # 1. 初始化能到达的最远位置
    max_reach = 0
    n = len(nums)
    
    # 2. 遍历每个位置
    for i in range(n):
        # 3. 如果当前位置无法到达，直接返回 false
        if i > max_reach:
            return False
        
        # 4. 更新能到达的最远位置
        max_reach = max(max_reach, i + nums[i])
        
        # 5. 如果已能到达最后位置，返回 true
        if max_reach >= n - 1:
            return True
    
    return True

```

## 易错点
1. **边界检查**：i > max_reach 说明当前位置无法到达
2. **最远位置的更新**：每次都要用 max(max_reach, i + nums[i]) 保持最远距离
3. **提前返回**：当 max_reach >= n-1 时可以提前返回 true，优化性能
4. **初始化**：max_reach 初始化为 0（从位置 0 开始，能到达位置 0）
5. **边界元素**：特殊处理只有一个元素的情况（已在初始化中处理）
