# 121. 买卖股票的最佳时机

## 题目
给定一个数组 prices ，其中 prices[i] 是一支给定股票第 i 天的价格。

你只能选择 某一天 买入这只股票，并选择 在未来的某一个不同的日子 卖出该股票。设计一个算法来计算你能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

示例:
- 输入: prices = [7,1,5,3,6,4]
  输出: 5
  解释: 在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5
  
- 输入: prices = [7,6,4,3,2,1]
  输出: 0
  解释: 在这种情况下，没有交易完成，所以最大利润为 0
  
- 输入: prices = [2,4,1,7,5,11]
  输出: 9
  解释: 买入价格为 2，卖出价格为 11，利润 = 11 - 2 = 9

## 思路
使用**贪心算法**，一次遍历求解：
1. 维护当前为止的最小价格（min_price）
2. 对于每个价格，计算从最小价格卖出的利润
3. 更新最大利润
4. 贪心策略：买入尽可能低的价格，卖出尽可能高的价格
5. 时间复杂度：O(n)，空间复杂度：O(1)

关键：不需要两层循环遍历所有买卖组合，一次遍历维护最小值和最大利润即可

## Python 代码

```python
def maxProfit(prices):
    # 1. 如果价格数量少于 2，无法完成买卖
    if not prices or len(prices) < 2:
        return 0
    
    # 2. 初始化最小价格和最大利润
    min_price = prices[0]
    max_profit = 0
    
    # 3. 遍历每个价格
    for price in prices[1:]:
        # 4. 计算当前价格卖出的利润
        profit = price - min_price
        # 5. 更新最大利润
        max_profit = max(max_profit, profit)
        # 6. 更新最小价格
        min_price = min(min_price, price)
    
    return max_profit

```

## 易错点
1. **贪心策略的正确性**：维护最小价格和最大利润足够求解，不需要动态规划
2. **起始条件**：需要检查数组长度，至少需要 2 个元素才能完成交易
3. **初始化顺序**：先取 prices[0] 作为最小价格，从 prices[1] 开始遍历
4. **利润计算**：当前价格减去历史最低价格，即 price - min_price
5. **最大利润更新**：每次都要更新最小价格，以保证后续计算正确
