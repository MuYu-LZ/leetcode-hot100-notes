# 076. 最小覆盖子串

## 题目
给你一个字符串 s 、一个字符串 t 。返回 s 中涵盖 t 所有字符的最小子串。如果 s 中不存在涵盖 t 所有字符的子串，则返回空字符串 ""。

注意：
- 对于 t 中重复字符，我们寻求的子串中该字符的个数应不少于 t 中该字符的个数。
- 如果 s 中存在这样的子串，我们保证它是唯一的答案。

示例 1:
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小窗口子串 "BANC" 包含来自字符串 t 的 "A"、"B" 和 "C"。

示例 2:
输入：s = "a", t = "aa"
输出：""
解释：t 中两个字符 'a' 均应包含在 s 的子串中，因此没有符合条件的子串，返回空字符串。

## 思路
该题使用**滑动窗口**结合**字符频率**统计：
1. 创建目标字符串 t 的频率表，以及当前窗口的频率表
2. 使用双指针维护一个可变大小的滑动窗口
3. 右指针逐个扩展窗口，直到包含 t 的所有字符
4. 当窗口有效时，左指针逐个收缩，尽量减小窗口大小，同时保持包含所有目标字符
5. 记录过程中满足条件且长度最小的子串

关键思想：先扩展找到包含所有字符的窗口，再收缩找到最小的有效窗口。

## Python 代码

```python
class Solution:
    def minWindow(self, s: str, t: str) -> str:
        # 1. 边界处理
        if not s or not t or len(s) < len(t):
            return ""
        
        # 2. 创建目标字符串的频率表
        t_count = {}
        for char in t:
            t_count[char] = t_count.get(char, 0) + 1
        
        # 3. 初始化滑动窗口变量
        window_count = {}
        left = 0
        min_len = float('inf')
        min_start = 0
        required = len(t_count)  # 需要覆盖的不同字符数
        formed = 0  # 当前窗口中满足频率要求的字符数
        
        # 4. 右指针扩展窗口
        for right in range(len(s)):
            # 5. 将右指针字符加入窗口
            char = s[right]
            window_count[char] = window_count.get(char, 0) + 1
            
            # 6. 如果该字符的频率达到目标频率，增加 formed 计数
            if char in t_count and window_count[char] == t_count[char]:
                formed += 1
            
            # 7. 当窗口包含所有目标字符时，尝试收缩左边界
            while left <= right and formed == required:
                char = s[left]
                
                # 8. 记录最小窗口
                if right - left + 1 < min_len:
                    min_len = right - left + 1
                    min_start = left
                
                # 9. 移除左指针字符
                window_count[char] -= 1
                if char in t_count and window_count[char] < t_count[char]:
                    formed -= 1
                
                left += 1
        
        return "" if min_len == float('inf') else s[min_start:min_start + min_len]

```

## 易错点
1. **formed 计数的含义**：记录的是满足频率要求的不同字符数，而非总字符个数
2. **收缩的时机**：只有当 formed == required（窗口包含所有目标字符）时，才开始收缩左边界
3. **频率更新的顺序**：先加入右端字符，再更新 formed；移除左端字符后再更新 formed
4. **最小窗口的记录**：在收缩过程中（即窗口有效时）记录长度最小的子串，这样能找到全局最优解
