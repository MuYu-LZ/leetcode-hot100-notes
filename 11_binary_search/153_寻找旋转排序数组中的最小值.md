# 153. 寻找旋转排序数组中的最小值

## 题目
已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 [0,1,2,4,5,6,7] 在旋转后可能变为 [4,5,6,7,0,1,2] 。

给你一个元素值 互不相同 的数组 nums ，它原本是一个升序排列的数组，并按上述情形进行了旋转。请你找出并返回数组中的最小元素。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

示例:
- 输入: nums = [3,4,5,6,7,0,1,2]
  输出: 0
  
- 输入: nums = [2,1]
  输出: 1
  
- 输入: nums = [1,2]
  输出: 1

## 思路
使用**二分查找**找旋转点：
1. 旋转数组可分为两个升序段：左段（大值）和右段（小值）
2. 最小值要么在左段的末尾，要么在右段的开头
3. 与右边界元素比较（nums[right]）：
   - 如果 nums[mid] > nums[right]，最小值在右侧，left = mid + 1
   - 如果 nums[mid] < nums[right]，最小值在左侧（包括 mid），right = mid
4. 关键：使用右边界而非中间值作为比较基准

关键：每次比较后能确定最小值必然在的一侧，最终收敛到最小值

## Python 代码

```python
def findMin(nums):
    # 1. 初始化二分查找的边界
    left, right = 0, len(nums) - 1
    
    # 2. 循环进行二分查找
    while left < right:
        # 3. 计算中点
        mid = (left + right) // 2
        
        # 4. 将中点与右边界比较
        if nums[mid] > nums[right]:
            # 5. 最小值在右侧，left 更新为 mid + 1
            left = mid + 1
        else:
            # 6. 最小值在左侧（包括 mid），right 更新为 mid
            right = mid
    
    # 7. 最终 left == right，该位置就是最小值
    return nums[left]

```

## 易错点
1. **与右边界比较**：一定要与 nums[right] 比较，而不是 nums[mid] 或 nums[left]
2. **mid > right 的处理**：表示最小值在右侧（旋转点在左侧），left = mid + 1
3. **mid < right 的处理**：表示最小值在左侧或就是 mid，right = mid（不能是 mid - 1）
4. **循环条件**：使用 while left < right，当相等时循环结束，left/right 指向最小值
5. **与有重复元素的区别**：这题元素互不相同，所以不需要处理 nums[mid] == nums[right] 的情况
