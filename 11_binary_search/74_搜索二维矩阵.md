# 74. 搜索二维矩阵

## 题目
编写一个高效的算法来搜索 m x n 矩阵 matrix 中的一个目标值 target 。该矩阵具有以下特性：
- 每行的整数从左到右按升序排列。
- 每列的整数从上到下按升序排列。

示例:
- 输入: matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 3
  输出: true
  
- 输入: matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 13
  输出: false
  
- 输入: matrix = [[1]], target = 1
  输出: true

## 思路
**方法一：从右上角或左下角开始搜索**（O(m + n)）：
1. 从右上角 (0, n-1) 开始
2. 如果 matrix[row][col] == target，返回 true
3. 如果 matrix[row][col] > target，col 递减（向左）
4. 如果 matrix[row][col] < target，row 递增（向下）

**方法二：二分查找**（O(log(m × n))）- 推荐：
1. 将二维矩阵看作一维有序数组
2. 索引 mid 对应的矩阵位置为：row = mid // n，col = mid % n
3. 使用标准二分查找算法

关键：利用矩阵的有序性质，选择合适的搜索策略

## Python 代码

```python
def searchMatrix(matrix, target):
    # 1. 检查矩阵是否为空
    if not matrix or not matrix[0]:
        return False
    
    m, n = len(matrix), len(matrix[0])
    
    # 2. 将二维矩阵看作一维有序数组进行二分查找
    left, right = 0, m * n - 1
    
    # 3. 标准二分查找
    while left <= right:
        mid = (left + right) // 2
        # 4. 将一维索引转换为二维坐标
        row = mid // n
        col = mid % n
        
        # 5. 获取中间值
        mid_value = matrix[row][col]
        
        # 6. 与目标值比较
        if mid_value == target:
            return True
        elif mid_value < target:
            # 7. 目标在右侧
            left = mid + 1
        else:
            # 8. 目标在左侧
            right = mid - 1
    
    return False

```

**方法二的替代实现（从右上角开始）**：
```python
def searchMatrix_v2(matrix, target):
    # 1. 从右上角开始
    if not matrix or not matrix[0]:
        return False
    
    row, col = 0, len(matrix[0]) - 1
    
    # 2. 遍历矩阵
    while row < len(matrix) and col >= 0:
        if matrix[row][col] == target:
            # 3. 找到目标
            return True
        elif matrix[row][col] > target:
            # 4. 当前值太大，向左移动
            col -= 1
        else:
            # 5. 当前值太小，向下移动
            row += 1
    
    return False
```

## 易错点
1. **一维到二维的转换**：row = mid // n，col = mid % n 的对应关系要正确
2. **空矩阵处理**：需要检查 matrix 和 matrix[0] 是否为空
3. **边界检查**：二分查找的 left 和 right 初始值应该是 0 和 m*n-1
4. **时间复杂度的选择**：二分查找 O(log(m*n)) 比遍历 O(m + n) 更优
5. **起点的选择**：从右上角或左下角开始遍历都可以，关键是每步能消除一行或一列
