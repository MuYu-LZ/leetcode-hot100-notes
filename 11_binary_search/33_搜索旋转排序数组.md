# 33. 搜索旋转排序数组

## 题目
整数数组 nums 在预先未知的某个点上进行了旋转（例如， [0,1,2,4,5,6,7] 经旋转后可能成为 [4,5,6,7,0,1,2] ）。

给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，返回它的索引，否则返回 -1 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

示例:
- 输入: nums = [4,5,6,7,0,1,2], target = 0
  输出: 4
  
- 输入: nums = [4,5,6,7,0,1,2], target = 3
  输出: -1
  
- 输入: nums = [1], target = 1
  输出: 0

## 思路
使用**二分查找**，关键是判断哪一侧是**有序的**：
1. 初始化 left = 0, right = len(nums) - 1
2. 在循环中：
   - 计算 mid = (left + right) // 2
   - 如果 nums[mid] == target，返回 mid
   - 判断左半边还是右半边是有序的：
     - 如果 nums[left] <= nums[mid]，左半边有序
     - 否则，右半边有序
   - 根据有序的一侧判断 target 在哪里
   - 调整 left 或 right
3. 返回 -1（未找到）

关键：不是直接比较两个值，而是先确定哪一侧是有序的，再判断 target 在不在这一侧

## Python 代码

```python
def search(nums, target):
    # 1. 二分查找的初始化
    left, right = 0, len(nums) - 1
    
    # 2. 循环进行二分查找
    while left <= right:
        mid = (left + right) // 2
        
        # 3. 如果找到目标值，直接返回
        if nums[mid] == target:
            return mid
        
        # 4. 判断左半边是否有序
        if nums[left] <= nums[mid]:
            # 5. 左半边有序，检查 target 是否在左侧范围内
            if nums[left] <= target < nums[mid]:
                # 6. target 在左侧，继续在左侧查找
                right = mid - 1
            else:
                # 7. target 不在左侧，在右侧查找
                left = mid + 1
        else:
            # 8. 右半边有序，检查 target 是否在右侧范围内
            if nums[mid] < target <= nums[right]:
                # 9. target 在右侧，继续在右侧查找
                left = mid + 1
            else:
                # 10. target 不在右侧，在左侧查找
                right = mid - 1
    
    return -1

```

## 易错点
1. **有序侧的判断**：关键是用 <= 比较判断有序侧，不是用 < 或 >
2. **target 的范围判断**：需要同时检查目标值是否在有序侧的实际范围内（nums[left] 到 nums[right]）
3. **边界条件**：target 的范围检查中，左侧用 <=，右侧用 <，避免边界错误
4. **中间值相等**：nums[mid] == target 必须单独处理，不能合并到有序侧判断
5. **旋转点的存在**：即使数组旋转过，至少有一侧是完全有序的，利用这一性质简化问题
