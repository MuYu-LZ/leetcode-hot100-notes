# 215. 数组中的第K个最大元素

## 题目
给定整数数组 nums 和整数 k，请返回数组中第 k 个最大的元素。

请注意，你需要找的是数组排序后的第 k 个最大的元素，而不是第 k 个不同的元素。

你必须设计并实现时间复杂度为 O(n) 的算法解决此问题。

示例:
- 输入: nums = [3,2,1,5,6,4], k = 2
  输出: 5
  
- 输入: nums = [3,2,3,1,2,4,5,5,6], k = 4
  输出: 4
  
- 输入: nums = [1], k = 1
  输出: 1

## 思路
使用**最小堆**维护 k 个最大的元素：
1. 维护一个大小为 k 的最小堆
2. 遍历数组：
   - 如果堆大小小于 k，直接添加元素
   - 否则，如果当前元素大于堆顶（最小），则删除堆顶，添加当前元素
3. 最后堆顶就是第 k 大元素
4. 时间复杂度：O(n log k)（可进一步优化到 O(n) 使用快选择算法）

关键：最小堆的堆顶始终是当前 k 个最大元素中的最小值

## Python 代码

```python
import heapq

def findKthLargest(nums, k):
    # 1. 创建大小为 k 的最小堆
    heap = []
    
    # 2. 遍历数组
    for num in nums:
        if len(heap) < k:
            # 3. 堆未满，直接添加
            heapq.heappush(heap, num)
        elif num > heap[0]:
            # 4. 堆已满且当前数比堆顶大
            # 5. 删除堆顶（最小值），添加当前数
            heapq.heapreplace(heap, num)
    
    # 6. 堆顶是第 k 大元素
    return heap[0]

```

## 易错点
1. **最小堆 vs 最大堆**：使用最小堆维护 k 个最大元素，堆顶是最小的，便于淘汰
2. **堆大小限制**：堆保持大小 k，这样第 k 大元素就是堆顶
3. **heapreplace 的作用**：一次性弹出并压入，比 heappop 后 heappush 更高效
4. **比较条件**：只有当新元素**严格大于**堆顶时才替换，等于时不需要替换
5. **时间复杂度**：O(n log k)，当 k 远小于 n 时优于排序的 O(n log n)
