# 287. 寻找重复数

## 题目
给定一个包含 n + 1 个整数的数组 nums，其数字都在 n 到 1 之间（包含 n 和 1）。

请找出一个重复的数（不一定唯一），要求：
- 不能修改数组（每个元素只能读一次）
- 只能使用 O(1) 的额外空间
- 时间复杂度不要超过 O(n²)

示例:
- 输入: nums = [1,3,4,2,2]
  输出: 2
  
- 输入: nums = [3,1,3,4,2]
  输出: 3
  
- 输入: nums = [2,5,9,6,4,3,7,8,4,7]
  输出: 4 或 7

## 思路
**方法一：哈希表**（不符合 O(1) 空间要求，但直观）：
记录已见过的数，遇到重复则返回。

**方法二：排序**（修改了数组）：
排序后相邻相同的元素就是重复的数。

**方法三：二分查找**（O(n log n) 时间，O(1) 空间）：
- 根据鸽笼原理，如果有 x 个数小于等于 mid，则：
  - 如果 x <= mid，重复数在 mid 右侧
  - 否则，重复数在 mid 左侧

**方法四：Floyd 循环检测**（O(n) 时间，O(1) 空间）- 推荐：
将数组视为链表，nums[i] 的下一个节点是 nums[nums[i]]。
重复的数对应于链表中的环，用快慢指针找环的入口。

关键：将数组问题转化为链表环检测问题

## Python 代码

```python
# 方法三：二分查找
def findDuplicate_binary(nums):
    # 1. 初始化二分查找的范围
    left, right = 1, len(nums) - 1
    
    # 2. 二分查找
    while left < right:
        mid = (left + right) // 2
        
        # 3. 计算小于等于 mid 的数的个数
        count = sum(1 for num in nums if num <= mid)
        
        # 4. 根据个数判断重复数的位置
        if count <= mid:
            # 5. 重复数在 mid 右侧
            left = mid + 1
        else:
            # 6. 重复数在 mid 左侧
            right = mid
    
    return left

```

**方法四：Floyd 循环检测（最优）**：
```python
def findDuplicate(nums):
    # 1. 初始化快慢指针（从索引 0 开始）
    slow, fast = 0, 0
    
    # 2. 快慢指针相遇，找到环内的某个点
    while True:
        slow = nums[slow]
        fast = nums[nums[fast]]
        if slow == fast:
            break
    
    # 3. 找到环的入口（重复数）
    # 从 0 和相遇点同时出发，每次移动一步
    slow = 0
    while slow != fast:
        slow = nums[slow]
        fast = nums[fast]
    
    return slow

```

## 易错点
1. **问题转化**：关键是理解为什么可以将数组问题转化为链表环检测
2. **数组索引与值的映射**：nums[i] 表示下一个节点的索引
3. **环的存在**：由于数字范围和数组大小的关系，重复数必然导致环
4. **Floyd 算法**：第一次相遇不是环的入口，需要第二次从头开始找入口
5. **二分查找的条件**：count <= mid 时搜索右半部分，这与通常的二分不同
