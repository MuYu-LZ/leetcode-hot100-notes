# 128. 最长连续序列 

## 题目
给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

 

示例 1：

输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。



## 思路
该题核心思路是利用哈希集合（Set）快速查询，避免排序带来的时间复杂度。算法步骤为：
1. 将所有数字放入集合中，利用集合的 O(1) 查询特性
2. 遍历集合中的每个数字，只从某个连续序列的**起点**开始计数（通过检查 `x-1` 是否在集合中来判断）
3. 从起点开始，逐个检查 `x+1, x+2, ...` 是否在集合中，计算连续序列的长度
4. 记录最长的连续序列长度

## Python 代码

```python
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        # 1. 将数组转换为集合，提供 O(1) 的查询时间
        st = set(nums)
        ans = 0
        
        # 2. 遍历集合中的每个数字
        for x in st:
            # 3. 如果 x-1 存在于集合中，说明 x 不是序列的起点，跳过
            if x - 1 in st:
                continue
            
            # 4. 从当前数字开始，检查连续的下一个数字
            y = x + 1
            while y in st:
                y += 1
            
            # 5. 计算连续序列的长度，更新最大值
            ans = max(ans, y - x)
        
        return ans

```

## 易错点
1. **只从序列起点开始计数**：必须检查 `x-1` 是否在集合中。如果在，说明 x 不是起点，要跳过，否则会重复计数
2. **集合而非列表**：使用集合的 O(1) 查询性能至关重要，保证整体时间复杂度为 O(n)
