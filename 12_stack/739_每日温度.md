# 739. 每日温度

## 题目
给定一个整数数组 temperatures ，表示每天的温度，返回一个数组 answer ，其中 answer[i] 是指在第 i 天之后，下一个更高温度出现在几天后。如果气温在这之后都不会升高，请在该位置用 0 来代替。

示例:
- 输入: temperatures = [73,74,75,71,69,72,76,73]
  输出: [1,1,4,2,1,1,0,0]
  
- 输入: temperatures = [30,40,50,60]
  输出: [1,1,1,0]
  
- 输入: temperatures = [30,60,90]
  输出: [1,1,0]

## 思路
使用**单调栈**解决：
1. 栈存储**索引**，确保栈内对应的温度是递减的
2. 遍历数组，对于每个温度：
   - 如果当前温度大于栈顶对应的温度，则弹出栈顶，计算距离
   - 继续弹出直到栈为空或当前温度小于等于栈顶
   - 将当前索引压入栈
3. 这样每个元素只被处理一次，时间复杂度 O(n)

关键：栈内的索引对应的温度形成递减序列，当遇到更高温度时立即处理

## Python 代码

```python
def dailyTemperatures(temperatures):
    # 1. 初始化结果和单调栈
    n = len(temperatures)
    answer = [0] * n
    stack = []
    
    # 2. 遍历每一天的温度
    for i in range(n):
        # 3. 当前温度高于栈顶对应的温度时
        while stack and temperatures[i] > temperatures[stack[-1]]:
            # 4. 弹出栈顶索引
            prev_idx = stack.pop()
            # 5. 计算天数差
            answer[prev_idx] = i - prev_idx
        
        # 6. 将当前索引压入栈
        stack.append(i)
    
    return answer

```

## 易错点
1. **栈存储的是索引而非值**：需要索引来计算天数差，即 i - prev_idx
2. **单调栈的单调性**：栈内温度必须保持递减，违反时需要弹出
3. **while 循环条件**：必须是 while 而非 if，可能有多个更低的温度需要一起处理
4. **结果初始化**：未找到更高温度的位置默认为 0，直接初始化数组为 0
5. **时间复杂度**：虽然有嵌套循环，但每个元素最多被压入和弹出一次，总体 O(n)
