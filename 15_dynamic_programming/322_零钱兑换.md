# 322. 零钱兑换

## 题目
给你一个整数数组 coins ，表示不同面额的硬币；以及一个整数 amount ，表示总金额。

计算并返回可以凑成总金额所需的 **最少的硬币个数** 。如果无法凑成总金额，返回 -1 。

你可以认为每种硬币的数量是无限的。

示例:
- 输入: coins = [1,2,5], amount = 5
  输出: 5
  解释: 5=5（1 种硬币），或 5=2+2+1（3 种硬币），最少需要 1 枚硬币
  
- 输入: coins = [2], amount = 3
  输出: -1
  
- 输入: coins = [10], amount = 10
  输出: 1

## 思路
使用**动态规划**，完全背包思想：
1. dp[i] 表示凑成金额 i 所需的最少硬币个数
2. 对于每个金额 i，尝试使用每种硬币：
   - 若使用硬币 coin，则转化为凑成 (i - coin) 的子问题
   - 递推关系：dp[i] = min(dp[i], dp[i - coin] + 1)
3. 初始条件：dp[0] = 0（凑成 0 需要 0 枚硬币）
4. 无法凑成则返回 -1
5. 遍历顺序：外层 amount，内层 coins（完全背包）

关键：每枚硬币可重复使用，所以是完全背包问题

## Python 代码

```python
def coinChange(coins, amount):
    # 1. 初始化 dp 数组，dp[i] = 凑成金额 i 的最少硬币个数
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0  # 2. 初始条件：凑成 0 需要 0 枚硬币
    
    # 3. 遍历从 1 到 amount 的每个金额
    for i in range(1, amount + 1):
        # 4. 遍历每种硬币
        for coin in coins:
            # 5. 如果硬币面额不超过金额
            if coin <= i:
                # 6. 尝试使用该硬币，更新最少硬币个数
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    # 7. 如果 dp[amount] 仍是无穷大，说明无法凑成
    return dp[amount] if dp[amount] != float('inf') else -1

```

## 易错点
1. **初始化**：dp[0] = 0，其他元素初始化为无穷大
2. **递推关系**：dp[i] = min(dp[i], dp[i - coin] + 1)
3. **硬币范围检查**：coin <= i 才能考虑使用该硬币
4. **遍历顺序**：外层 amount 内层 coins（完全背包）
5. **返回值**：无法凑成时返回 -1，需要检查 dp[amount] 是否为无穷大
